# Project Workflow

1. Idea: capture the founder or user intent.
2. PRD: define problem, users, scope, and acceptance criteria.
3. Architecture: identify boundaries, data model, APIs, and risks.
4. Task breakdown: split backend, frontend, data, integration, docs, and tests.
5. Implementation: make focused changes following engineering standards.
6. Test: run formatters, automated tests, and manual acceptance checks.
7. Review: review code, architecture, security, and risk.
8. Approval: request approval for high-risk actions.
9. Release: perform approved release steps with rollback awareness.
10. Retrospective: record lessons, incidents, and follow-up tasks.

Each stage should leave enough written context for another engineer or agent to continue safely.
