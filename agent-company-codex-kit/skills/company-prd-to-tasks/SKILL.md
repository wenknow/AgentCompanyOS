# Company PRD To Tasks

Convert product input into an implementation-ready plan with:

1. Product goal.
2. User roles.
3. Core user stories.
4. MVP scope.
5. Non-MVP scope.
6. API, database, frontend, backend, integration, and test task breakdown.
7. Acceptance criteria.
8. Risks and dependencies.
9. Phased development plan.

Keep scope explicit. Mark assumptions, defer nonessential work, and call out approval needs for high-risk features.
